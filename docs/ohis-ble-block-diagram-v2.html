OHIS Interface

User-side 8P8C RJ-45. Three grounds (MicGND pin 3, HPGND pin 5, PwrGND pin 7) kept separate on PCB. Vcc from pin 1 feeds the MCP73871 VIN as a secondary power source when no USB present, but is not used for battery charging. PTT asserted by 2N7002 N-FET pulling pin 2 to PwrGND on GPIO command.

Audio Path

HP audio (0 to −10dBV) received through 10kΩ/10nF filter into NAU8822 ADC. ESP32 I²S master clocks the codec at 48kHz/24-bit. BT audio encoded to SBC/AAC for headsets or decoded from HFP SCO for mic. DAC output padded to −45dBV electret level through 600Ω output pad and 100nF DC block before OHIS pin 6.

Bluetooth

ESP32 Bluedroid stack runs A2DP Source + HFP Audio Gateway for Classic BT headsets simultaneously with BLE for ASHA hearing aids and BLE HID PTT switches. PTT switch button press triggers ESP32 HID callback in ~15ms, driving the OHIS PTT FET. Headset's own multifunction button can also be remapped to PTT via HFP call-answer event.

Power System

MCP73871 handles USB-C input, LiPo charging (500mA via Rprog), and power-path — system runs from USB while charging the battery with no external OR circuit needed. TPS63020 buck-boost maintains 3.3V across full LiPo discharge (4.2V→3.0V). Estimated 34h runtime on 2000mAh cell at typical BT-active load.

Sharp MIP + Frontlight

LS013B7DH03 draws ~5µA with image held in pixel memory — no refresh needed. EXTCOMIN driven by ESP32 timer at 1Hz. White PLCC-2 LED frontlight via BSS138 FET + PWM. GL5528 LDR on ADC pin controls duty automatically; any button press overrides to next brightness step with 30s auto-resume timeout.

PTT Complete Flow

Three independent PTT sources all funnel to the same 2N7002 gate: (1) BT HID switch over BLE/Classic BT, (2) headset multifunction button via HFP callback, (3) physical front-panel button wired direct to GPIO. Any source keys the radio. TX active shown on red status LED and optionally broadcast as BLE status to a companion phone app.
